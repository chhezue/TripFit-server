# TripFit MVP

> 1차 출시 범위. NotebookLM 기획 자료 정리본.

## MVP 목표

- **검증 가설**: 복잡한 근무/연차 조건을 반영한 일정 추천 기능이 실제 여행 확정률을 높이고 조율 시간을 단축시킬 것이다.
- **성공 지표**: 평균 일정 취합 및 확정 시간 단축, 추천 조회 대비 확정 전환율, 최종 여행 일정 확정률.

## In Scope (MVP 포함)

- [ ] 사용자 인증 및 소셜 로그인
- [ ] 여행방 생성 및 옵션 관리(이름, 기간, 인원 등)
- [ ] 초대 링크 생성 및 카카오톡 공유
- [ ] 참여자별 일정 응답 (오전/오후/저녁 단위, 미정 상태 포함)
- [ ] 개인 근무 패턴 및 연차 조건 입력
- [ ] 그룹 달력 시각화
- [ ] 추천 알고리즘 및 TOP 3 후보/근거 제공
- [ ] 최종 일정 확정 기능

## Out of Scope (MVP 제외, v2 이후)

- 가격(항공·숙소) 조건 반영 추천
- 사진 인식을 통한 일정 등록 기능
- 과거 데이터 기반 일정 자동 채우기
- 숙소/교통 예약 서비스 연계
- 방장의 참여자 강제보내기 및 권한 관리

## 우선순위

| 순위 | 기능 | 이유 |
|------|------|------|
| P0 | 여행방 생성/참여 및 일정 추천 | 제품의 핵심 가치인 빠른 의사결정 지원을 위한 필수 기능 |
| P0 | 근무/연차 조건 기반 일정 응답 | 타 서비스와 차별화되는 상세 조건 수집 기능 |
| P1 | 리마인드 알림 및 메시지 공유 | 일정 취합 시간을 단축하기 위한 주요 장치 |
| P1 | 그룹 달력 시각화 | 총대가 현황을 직관적으로 파악하기 위해 필요 |
| P2 | 소셜 계정 연동 및 알림 상세 설정 | 사용자 편의 기능으로 핵심 가치 검증 이후 고도화 가능 |

## MVP 완료 기준

방장이 설정한 추천 기준(참여 인원, 연차 등)에 따라 시스템이 후보를 생성하고, 방장이 그중 하나를 선택해 일정을 확정할 수 있는 프로세스가 단절 없이 작동해야 함.

## 의존성 / 선행 조건

- 외부 캘린더(구글, 네이버 등) API 연동은 기획상 `[제안]`이나, 일정 입력 피로도 감소를 위해 MVP에서 우선 검토 권장.
- 알림·리마인드 발송 주기·채널 상세 정책은 `[미정]` — 구현 전 기획 확인 필요.

## 기획 메모 (NotebookLM)

- **근거 약한 In Scope**: 외부 캘린더 연동 — 자료는 Could 수준이나 UX상 MVP 포함 강력 권장.
- **미정**: 리마인드 발송 주기·채널 상세.
