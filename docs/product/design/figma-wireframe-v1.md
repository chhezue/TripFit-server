## 핵심 도메인

### 1. 여행방 (Trip Room)

- 생성자 = **방장**
- 필수: 이름, 희망 기간(시작~종료), 희망 여행 일수(박/일), 참여 인원
- 여행지: **정했어요**(필수 입력) / **아직 못 정했어요**(선택 입력) 분기
- 생성 후 방장이 희망 기간 내 **일정 가용성** 입력 (드래그·캘린더)
- 홈: **진행중인 여행** 카드 + **전체 여행보기** 리스트
- 필터: **내가 생성한 방만 보기**
- 고정(pin): 클릭 시 해당 여행방 우선 조회
- 메뉴: **삭제하기**, **링크 공유하기**
- 상세 탭: **달력** | **추천 일정**
- 관리: 기간·박일·인원 수정, **여행방 삭제하기**, 멤버 관리

### 2. 여행방 상태 (목록·카드 뱃지)

| 상태 | 의미 (와이어프레임 기준) |
|------|--------------------------|
| 응답대기중 | 멤버 일정 응답 미완 |
| 조율중 | 응답 수집·조율 진행 |
| 일정 확정 | 최종 일정 확정됨 |

### 3. 일정 가용성 (멤버·날짜)

날짜별 표시 (범례):

| 표시 | 라벨 |
|------|------|
| 전원 가능 | 모두 가능 |
| 일부 제약 | 부분 가능 |
| 불가 | 완전 불가 |

생성 플로우 범례 (시간대): **불가능**, **아침불가**, **점심불가**, **저녁불가**, **불확실**

### 4. 응답·추천

- **응답률**: `N/M 응답` (예: 4/5)
- **응답 요청** 액션 (미응답자 독촉)
- 추천 전제: `현재 N명의 응답을 기반으로한 추천 일정`
- 추천 전략(카드):
  - **기본 선택** — 참석률과 연차 균형
  - **휴가 아끼기**
  - **모두 참석** — 전원 참석 가능 우선
  - **확실하게 가기** — 변수 적은 일정 우선
- 추천 결과: 순위, 기간(예: `6.12(월) - 6.15(목)`), **추천 근거**
- 액션: **추천 일정보기**, **일정 확정하기**, **추천 일정 공유하기**
- 피드백: `이 추천이 도움이 되었나요?`

### 5. 사용자·온보딩·계정

- 로그인: **Google**, **Kakao**, **Apple**
- 가입 후(선택): **구글 캘린더** 연동, **나중에 하기** (네이버 캘린더 **제외** — [`007`](../../decisions/007-user-profile-onboarding.md))
- **기본정보**: 성·이름(필수), 근무 일정(출근·퇴근·출근 요일·세부 근무 블록), 연차 조건
  - 최대 사용 가능 연차 수
  - 연차 사전 신청 기간
  - 반차 사용 가능 여부
  - 법정 공휴일 휴무 여부
- 마이페이지: 캘린더 연동, **알림 허용**, **탈퇴하기**
- 탈퇴 경고: `탈퇴하면 내가 만든 여행방 데이터가 모두 사라져요`

### 6. 알림 (예시 카피)

- 참여: `OO님이 ‘여행방명’에 참여했습니다`
- 추천: `‘여행방명’의 추천 일정을 확인해보세요`
- 변경: `방장 OOO님이 여행 일자를 변경했습니다`
- 리마인드: `N월달의 불확실 일정을 업데이트 해보세요`
- 미입력: `N월달에 표시된 일정이 하나도 없어요...`

## API 설계 시 암시되는 리소스 (초안)

| 리소스 | 설명 |
|--------|------|
| `User` | 소셜 계정, 프로필, 근무·연차 설정 |
| `TripRoom` | 여행방 메타, 상태, 방장, pin |
| `TripMember` | 방–사용자, 역할(방장/멤버) |
| `Availability` | 멤버×날짜(×시간대) 가용성 |
| `Recommendation` | 전략, 후보 일정, 순위, 근거 |
| `Notification` | 타입, 읽음, 관련 여행방 |
| `CalendarIntegration` | Google 연동 상태 (`isGoogleCalendarConnected`) |

ERD는 `docs/architecture/erd.md`에 NotebookLM·기획과 함께 정제.
