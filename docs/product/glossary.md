# TripFit 용어집

> Figma Wireframe v1 반영. NotebookLM으로 보강 예정.

| 용어 | 정의 | 비고 |
|------|------|------|
| 여행방 | 멤버가 모여 일정을 조율하는 단위 | Trip Room |
| 방장 | 여행방 생성자·관리 권한 보유 | 홈 카드에 뱃지 표시 |
| 진행중인 여행 | 홈 상단 강조 카드로 보이는 활성 여행방 | pin 시 우선 조회 |
| 응답률 | 일정 응답 완료 멤버 비율 | 예: 4/5 응답 |
| 응답대기중 | 멤버 가용 일정 응답이 아직 안 끝난 상태 | 여행방 상태 |
| 조율중 | 응답 수집·일정 조율 중 | 여행방 상태 |
| 일정 확정 | 최종 여행 일정이 확정된 상태 | 여행방 상태 |
| 가용성 | 날짜(·시간대)별 참석 가능 여부 | 모두 가능/부분 가능/완전 불가 |
| 추천 일정 | 멤버 응답·연차 등을 반영한 후보 여행 기간 | 전략별 복수 후보 |
| 추천 전략 | 기본 선택, 휴가 아끼기, 모두 참석, 확실하게 가기 | 추천 알고리즘 입력 |
| 희망 기간 | 여행 가능성을 탐색하는 날짜 범위 | 생성 시 필수 |
| 희망 여행 일수 | 원하는 박/일 수 | 생성 시 필수 |
| 기본정보 | 근무 일정·연차 조건 등 개인 제약 | 추천 입력 |
