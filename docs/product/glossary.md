# TripFit 용어집

> NotebookLM + Figma Wireframe v1 통합.

## 용어

| 용어 | 정의 | 비고 |
|------|------|------|
| TripFit | 본 서비스 명칭 (구 When We Meet) | - |
| 총대 | 여행 모임에서 일정 조율·의사결정을 주도하는 사람 | 방장과 동의어 |
| 방장 | 여행방 생성자·관리 권한 보유 | 총대, Figma 뱃지 표시 |
| 여행방 | 특정 여행을 위해 참여자들이 모여 일정을 조율하는 공간 | Trip Room |
| 여행 희망 기간 | 여행이 가능한 전체 후보 기간 | 생성 시 필수 |
| 희망 여행 일수 | 실제 여행 기간 (예: 2박 3일) | 생성 시 필수 |
| 미정 (불확실 일정) | 확정되지 않아 변동 가능성이 있는 개인 일정 | 면접 대기, 연차 승인 전 등 |
| 추천 근거 | 특정 후보 일정이 선정된 이유 | 참여 인원, 리스크 등 |
| 늦참 / 조기 귀가 | 여행 도중 합류하거나 먼저 돌아오는 케이스 | 추천 점수 반영 |
| 동심원 시각화 | 그룹 달력에서 참여자 가능 여부를 겹쳐 보여주는 방식 | - |
| 진행중인 여행 | 홈 상단 강조 카드로 보이는 활성 여행방 | pin 시 우선 조회 |
| 응답률 | 일정 응답 완료 멤버 비율 | 예: 4/5 응답 |
| 응답대기중 | 멤버 가용 일정 응답 미완 상태 | 여행방 상태 |
| 조율중 | 응답 수집·일정 조율 진행 중 | 여행방 상태 |
| 일정 확정 | 최종 여행 일정 확정된 상태 | 여행방 상태 |
| 가용성 | 날짜(·시간대)별 참석 가능 여부 | 모두 가능/부분 가능/완전 불가 |
| 추천 일정 | 멤버 응답·연차 등을 반영한 후보 여행 기간 | TOP 3 후보 |
| 추천 전략 | 기본 선택, 휴가 아끼기, 모두 참석, 확실하게 가기 | Figma UI 기준 |
| 기본정보 | 근무 일정·연차 조건 등 개인 제약 | 추천 입력 |

## 약어

| 약어 | 풀네임 | 설명 |
|------|--------|------|
| IA | Information Architecture | 서비스 정보 구조·화면 위계 |
| PRD | Product Requirements Document | 제품 요구사항 정의서 |
| MVP | Minimum Viable Product | 최소 기능 제품 |
| KPI | Key Performance Indicator | 핵심 성과 지표 |
| VoC | Voice of Customer | 고객 의견 |
| CTA | Call To Action | 행동 유도 UI 요소 |
| BR | Business Rule | `business-rules/` 규칙 ID 접두 |
