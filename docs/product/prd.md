# TripFit PRD

> NotebookLM 기획 자료 정리본. 상세 UI는 `design/figma-wireframe-v1.md` 참고.

## 1. 개요

- **제품 한 줄 소개:** 여러 사람의 조건을 반영해 모두가 납득할 수 있는 최적의 여행 일정을 추천하는 의사결정 지원 서비스
- **비전:** 가장 적합한 여행 날짜를 빠르게 결정하도록 돕고, 일정 조율 문제로 여행이 무산되지 않게 함

## 2. 문제 정의

- **번거로운 일정 취합:** 참여자의 응답을 기다리고 미응답자에게 독촉하며 현황을 계속 확인해야 하는 번거로움
- **의사결정의 어려움:** 연차, 근무 패턴, 불확실한 일정 등 복잡한 조건을 직접 비교하며 판단하기 어려움
- **총대(방장)의 부담:** 최종 일정을 선택하고 근거를 설명해야 하는 책임이 특정 개인에게 집중되어 피로도 증가

## 3. 타깃 사용자

- **주요 사용자 (방장/총대):** 친구 모임에서 여행 계획을 주도하며 일정 조율의 부담을 느끼는 사용자
- **보조 사용자 (참여자):** 여행 일정 조율에 참여하며 자신의 일정을 간편하게 공유하고 싶은 사용자. 초대 링크 진입 후 **소셜 로그인 필수** (비회원·게스트 없음 — BR-USER-002 확정)

## 4. 핵심 기능

- **여행방 생성 및 관리:** 여행명, 희망 기간, 인원 등을 설정하여 조율 공간 생성
- **일정 시각화 및 입력:** 오전/오후/저녁 단위로 가능/불가/미정 상태 입력 및 그룹 달력 가시화
- **개인 맞춤 조건 설정:** 근무 패턴(교대 근무 등) 및 연차 사용 조건(개수, 사전 신청 기간 등) 반영
- **최적 일정 추천 로직:** 방장이 선택한 **추천 모드**(기본 / 모두 참석 / 휴가 아끼기 / 확실하게 가기)에 따라 가중치 기반 TOP 3 후보와 추천 근거 제공 (wave 2 전 모드 구현 — BR-TRIP-005)
- **일정 확정 및 공유:** 추천된 후보 중 하나를 선택하여 최종 일정을 확정하고 외부 공유

## 5. 사용자 시나리오 / 플로우

1. **방장:** 여행방 생성 및 초대 링크 생성
2. **참여자:** 링크 접속 → **소셜 로그인** → 본인 일정/근무 조건 입력
3. **방장/참여자:** 그룹 달력에서 실시간 응답 현황 확인 및 미응답자 리마인드
4. **방장:** 추천 모드(기본 / 모두 참석 / 휴가 아끼기 / 확실하게 가기) 선택 후 후보 조회
5. **방장:** 최종 일정 확정 및 결과 공유

## 6. 화면 / API 개요

- **홈 (내 여행):** **진행 중인 여행**(캐러셀·Pin) + **전체 여행 보기**(필터: 조율 중/확정·내가 생성한 방), 새 여행방 생성 진입 — D5 [`trip-room-api.md`](../specs/trip-room-api.md)
- **여행방 상세 (그룹 달력):** 날짜별 참여 가능 인원 분포 시각화, 참여자 응답 상태 확인
- **내 일정 관리:** 캘린더 연동 또는 직접 드래그를 통한 시간대별 일정 입력
- **추천 결과 페이지:** 추천 모드 선택 및 후보 일정 TOP 3 비교, 확정 버튼
- **설정/마이페이지:** 알림 설정, 소셜 계정 연동, 계정 관리

## 7. 비기능 요구사항

- **사용성:** 앱 설치 없이 웹 랜딩·딥링크 진입 지원 (로그인은 필수)
- **안정성:** 추천 로직의 엣지 케이스 대응(동점자 처리 규칙 등) `[미정]` 상세
- **확장성:** 향후 숙소/교통 예약 연계 및 비용 부담도 반영 고려

## 8. 제약 및 가정

- 방장은 1명만 존재하며 일정 확정 권한을 가짐
- 여행 희망 기간은 필수 입력 사항이며 추천 탐색 범위로 사용됨
- 개별 일정 명칭은 비공개로 하며 상태(가능/불가/미정)만 노출함
- 참여자는 비회원·게스트 없이 반드시 소셜 로그인 계정으로 참여 (BR-USER-002)
