# Trip Room (여행방) 비즈니스 규칙

> 출처: Figma Wireframe v1. `[미정]`은 기획 확인 필요.

## BR-ROOM-001

- **조건:** 여행방 생성 요청
- **동작:** 필수 필드 검증 — 이름, 희망 기간(시작≤종료), 희망 여행 일수, 참여 인원
- **위반 시:** 400, 필드별 오류 메시지

## BR-ROOM-002

- **조건:** 여행지 입력 분기
- **동작:** `정했어요` → 여행지 필수 / `아직 못 정했어요` → 여행지 선택(미입력 허용)
- **위반 시:** 400

## BR-ROOM-003

- **조건:** 여행방 생성 완료
- **동작:** 생성자를 방장·멤버로 등록, 초기 상태 `응답대기중` [제안]
- **위반 시:** —

## BR-ROOM-004

- **조건:** 홈 목록 조회
- **동작:** pin된 여행방을 진행중 카드·목록 상단 우선 노출
- **위반 시:** —

## BR-ROOM-005

- **조건:** `내가 생성한 방만 보기` 필터 on
- **동작:** 요청 사용자가 방장인 여행방만 반환
- **위반 시:** —

## BR-ROOM-006

- **조건:** 여행방 상태 전이
- **동작:** `응답대기중` → `조율중` → `일정 확정` (세부 전이 조건 `[미정]`)
- **위반 시:** 409, 잘못된 상태 전이

## BR-ROOM-007

- **조건:** 방장이 여행방 삭제
- **동작:** 여행방 및 관련 데이터 삭제 [미정: soft delete 여부]
- **위반 시:** 403, 방장 아님

## BR-ROOM-008

- **조건:** 링크 공유
- **동작:** 초대/공유 링크 발급 [미정: 만료·권한]
- **위반 시:** —
