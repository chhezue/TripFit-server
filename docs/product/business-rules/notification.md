# Notification (알림) 비즈니스 규칙

> NotebookLM 기획 자료 정리본. MVP 푸시·카카오 공유는 **wave 3**. 스케줄러 정기 발송은 **wave 4**.

| 규칙 ID | 규칙명 | 조건 | 동작 | 위반 시 (에러/제약) |
| :--- | :--- | :--- | :--- | :--- |
| **BR-NOTI-001** | 참여 완료 알림 | 참여자 **`POST /trips/join` 성공** (또는 동등 RESPONDED) | 방장에게 푸시 | wave 3 |
| **BR-NOTI-002** | 전원 응답 완료 알림 | `joinedMemberCount`가 목표에 도달·전원 RESPONDED 등 (세부 wave 3) | 방장에게 추천 유도 | wave 3 |
| **BR-NOTI-003** | 여행 정보 변경 알림 | 방장이 기간·일수·여행지 등 수정 시 (BR-TRIP-009·010) | 모든 참여자에게 변경 알림 | BR-USER-005 Off 시 미발송 |
| **BR-NOTI-004** | 일정 확정·취소 알림 | 방장이 확정 또는 취소 시 (BR-TRIP-007) | 모든 참여자에게 결과 알림 | BR-USER-005 Off 시 미발송 |
| **BR-NOTI-005** | 정기 리마인드 (스케줄러) | 매월 1일·15일 등 **정기** | 전체(또는 대상) 사용자 월 일정 업데이트 리마인드 | **wave 4** — MVP Out |
