# Schedule (일정·가용성) 비즈니스 규칙

> 출처: Figma Wireframe v1.

## BR-SCHEDULE-001

- **조건:** 방장이 여행방 생성 후 일정 입력
- **동작:** 희망 기간 내 날짜별 가용성 수집 (드래그/캘린더 UI)
- **위반 시:** 400, 기간 밖 날짜

## BR-SCHEDULE-002

- **조건:** 날짜 가용성 표시
- **동작:** 집계 범례 — `모두 가능` / `부분 가능` / `완전 불가`
- **위반 시:** —

## BR-SCHEDULE-003

- **조건:** 멤버 개인 일정 입력
- **동작:** 시간대 구분 지원 — 불가능, 아침불가, 점심불가, 저녁불가, 불확실
- **위반 시:** 400, 허용되지 않은 값

## BR-SCHEDULE-004

- **조건:** 응답률 계산
- **동작:** `응답 완료 멤버 수 / 전체 멤버 수` (예: 4/5)
- **위반 시:** —

## BR-SCHEDULE-005

- **조건:** 응답 요청
- **동작:** 미응답 멤버에게 알림·독촉 [미정: 재요청 쿨다운]
- **위반 시:** 403, 권한 없음

## BR-SCHEDULE-006

- **조건:** 외부 캘린더 연동 (Google/Naver)
- **동작:** 연동된 일정을 가용성 입력 보조 [미정: 동기화 주기·충돌 해결]
- **위반 시:** 502, 연동 실패

## BR-SCHEDULE-007

- **조건:** 개인 근무 일정·연차 설정
- **동작:** 출퇴근, 요일별 근무 블록, 최대 연차, 사전 신청 기간, 반차 가능, 공휴일 휴무 여부 저장
- **위반 시:** 400
