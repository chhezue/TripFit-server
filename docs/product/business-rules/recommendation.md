# Recommendation (추천 일정) 비즈니스 규칙

> 출처: Figma Wireframe v1.

## BR-REC-001

- **조건:** 추천 일정 생성
- **동작:** 최소 1명 이상 응답 필요; UI 카피 기준 `N명의 응답을 기반` [미정: 최소 N]
- **위반 시:** 422, 응답 부족

## BR-REC-002

- **조건:** 추천 전략 선택
- **동작:** 지원 전략 — `기본 선택`, `휴가 아끼기`, `모두 참석`, `확실하게 가기`
- **위반 시:** 400, 알 수 없는 전략

## BR-REC-003

- **조건:** 추천 결과 표시
- **동작:** 순위, 기간(시작~종료), `추천 근거` 텍스트 포함
- **위반 시:** —

## BR-REC-004

- **조건:** 일정 확정하기
- **동작:** 선택한 추천 후보로 여행방 상태 `일정 확정` 전이, 확정 일정 저장
- **위반 시:** 409, 이미 확정됨 / 403

## BR-REC-005

- **조건:** 추천 일정 공유
- **동작:** 멤버·외부 공유 가능한 payload 생성 [미정: URL vs 앱 내]
- **위반 시:** —

## BR-REC-006

- **조건:** 추천 피드백
- **동작:** `이 추천이 도움이 되었나요?` 응답 저장 [미정: 필수 여부]
- **위반 시:** —
