# 일정 확정 플로우

> NotebookLM 기획 자료 정리본.

- **목적:** 수집된 일정으로 추천 후보를 확인하고 최종 날짜 확정
- **액터:** 방장(총대)
- **사전 조건:** 최소 1명 이상 일정 제출 (전원 제출 권장 — BR-USER-007)

**단계:**

1. 「추천 일정 확인하기」 클릭
2. 추천 모드 선택 — 기본 / 모두 참석 / 휴가 아끼기 / 확실하게 가기 (BR-TRIP-005, BR-TRIP-011)
3. TOP 3 후보·근거 확인 (BR-TRIP-012 동점 시)
4. 후보 선택 또는 직접 날짜 입력
5. 「일정 확정하기」 (BR-TRIP-007)
6. 결과 공유 (카카오톡 등 — wave 3, BR-NOTI-008)

**성공 종료 조건:** `trip.status=CONFIRMED`, 참여자 알림 (BR-NOTI-004)

**예외 / 분기:**

- 확정 후 취소 → 취소 사유 입력 플로우 (MVP In)
- 참여자 확정 API 403 (BR-TRIP-007)
- 모두 참석 모드 후보 없음 (BR-TRIP-011)

**MVP 포함 여부:** In
