# 일정·조건 입력 플로우

> NotebookLM 기획 자료 정리본.

- **목적:** 제출된 일정·근무/연차 정보 수정 또는 온보딩 skip 사용자의 사전 조건 보완
- **액터:** 방장, 참여자
- **사전 조건:** 해당 여행방 참여 중 (`trip_member` 존재)

**단계:**

1. 여행방 상세 달력에서 「내 일정 입력/수정하기」 클릭
2. (선택) `schedule` CONDITION·AVAILABILITY 수정 (BR-TRIP-006, BR-USER-008)
3. AVAILABILITY 시간대 상태 변경 (BR-TRIP-002, BR-TRIP-003)
4. 「저장」 또는 「일정 제출하기」 (BR-USER-007)

**성공 종료 조건:** 그룹 달력·추천 입력 데이터 반영. 기간·일수 변경 시 추천 초기화 (BR-TRIP-010)

**예외 / 분기:**

- 본인 데이터만 수정 (BR-TRIP-004)
- 방장만 trip 메타(이름·기간·여행지) 수정 (BR-TRIP-009)
- 희망 기간 종료 후 여행방 수정 제한 `[미정]`
- 외부 캘린더 연동 — MVP `[제안]`, wave 4 후보
